OpenClaw から Hermes への移行は、もっと手間がかかるものだと思っていました。設定をどこまで引き継げるか分からず、認証情報まわりもやり直しになるかもしれないと見ていたからです。

実際にやってみると、印象はかなり違いました。結論から書くと、私のケースでは移行はかなり素直で、手で直したのは Codex の model 名くらいでした。しかも、引っ越した直後から、Slack での会話管理が良くなったことを体感できました。

この記事では、移行作業そのものと、移行してすぐ感じた運用上の差を分けて書きます。

## 移行はほぼ一発で済んだ

私としては、setup 周りの評価はかなり高いです。

一言で言うなら、**Codex の model 名だけ自動移行のあと手修正したが、ほかは認証情報含めコマンド一発で移行できて楽だった**、になります。

ここで良かったのは、単にファイルがコピーされたという話ではありません。認証情報や接続設定を含めて、日常運用に必要な土台がかなりそのまま引き継がれたことです。ゼロから組み直す感じではなく、既存の環境を素直に持ってこられた感覚がありました。

移行系の機能は、成功すると目立ちません。ですが本当は、目立たないこと自体がかなり大事です。移行のたびに「何が落ちたのか」「どの secret を入れ直せばよいのか」を掘る時間が必要だと、それだけで気持ちが削られます。今回の Hermes への移行では、その摩擦がかなり少なかったです。

## 唯一の手修正は Codex の model 名だった

完全にノータッチで終わったわけではありません。私の環境では、Codex の model 名だけは自動移行後に手で直しました。

ただ、ここは「大きな失敗があった」というより、移行結果の最終調整が一箇所だけ必要だった、という見え方です。認証が壊れたとか、Slack 側の接続を全部やり直したとか、そういう重い話ではありません。

むしろ、この一点だけで済んだので、全体としての印象はかなり良いままです。自動移行の品質は、完全無欠かどうかではなく、手修正がどれくらい局所的で済むかで決まるところがあります。その意味で、今回の移行は十分に楽でした。

## Hermes に移ってすぐ良かったのは Slack の会話管理

移行後にいちばん分かりやすく良かったのは、Slack での会話のまとまり方です。

今こうして Slack でやりとりしているこのスレッドも、Hermes はちゃんと一つの会話として扱えています。これは見た目の印象ではなく、実際にやりとりしていて感じる差です。話題が前後しても、同じ thread の文脈を続けて読めている感じがあります。

この点は、OpenClaw では少し不安がありました。同じチャンネルの中で複数スレッドが並んでいると、別の thread の会話が混ざっているように見える場面があり、会話の境界がやや怪しいことがありました。

もちろん、会話管理は外から完全に見えるものではありません。それでも、使っている側にはかなり分かります。文脈がずれた返答が減るか、話の継ぎ目が自然か、別スレッドの話を急に持ち込まないか。このあたりは、日常的に触っているとすぐ差が出ます。

Hermes には、後発らしいスマートさがあると感じています。派手な新機能というより、会話の境界をきちんと扱うような基礎体力が上がっている印象です。

## この差は「賢そう」より「任せやすい」に効く

エージェントを日常で使うときに大事なのは、単発で派手な答えを返すことより、会話を壊さないことだと思っています。

たとえば Slack の thread ごとに相談内容が違うとき、こちらが欲しいのは「広く覚えている感じ」ではなく、「今この thread の話だけをちゃんと続けてくれる感じ」です。別の会話が混ざると、それだけで確認コストが上がりますし、任せる側の集中も切れます。

その意味で、Hermes に移って最初に感じた価値は、賢さの誇示ではありませんでした。むしろ、**会話の境界が安定したことで、任せるときの信頼感が上がった**ことです。

これは地味ですが、かなり重要です。日々使う道具として見ると、こういう差の方があとから効いてきます。

## まとめ

OpenClaw から Hermes への移行は、私のケースではかなりスムーズでした。手で直したのは Codex の model 名くらいで、それ以外は認証情報を含めて素直に引き継げました。

そして、移行の価値は setup の楽さだけではありません。Slack のスレッドを一つの会話として扱う感じが明確に良くなり、日常運用での安心感も上がりました。

もし OpenClaw からの移行を迷っているなら、少なくとも私の感触では、試す価値は十分にあります。移行コストに対して、得られる改善は思ったより大きいです。
